nginx通过四层代理实现端口转发

# nginx通过四层代理实现端口转发

# 概念

四层代理工具（Layer 4 Proxy）是一种网络代理技术，它在传输层（Transport Layer）进行操作，代理网络连接，对网络流量进行转发和负载均衡。

四层代理工具通常用于 TCP 和 UDP 协议，例如代理 TCP 指向 Web 服务器的连接或者 UDP 数据流。

# 背景

公司原有的测试数据库在主机192.168.1.5上边，现在数据库转移到了192.168.1.4上， 在尽可能不改动其它配置的情况下的快速完成迁移需求，我能想到的合理的办法就是做一个四层代理，将原来请求到192.168.1.5的3306端口转发到192.168.1.4的3306端口。

通过官方文档，可以得知：Nginx中四层代理依赖模块 ngx_stream_core_module，该模块自 1.9.0 版开始可用。默认情况下，此模块不构建，应使用配置参数启用 --with-stream。

# 准备工作

  • 安装Nginx并启用--with-stream模块

安装步骤大致如下：

[root@linux-node1 src]# tar xf nginx-1.10.3.tar.gz 
[root@linux-node1 src]# cd nginx-1.10.3
[root@linux-node1 nginx-1.10.3]# useradd -s /sbin/nologin -M www
[root@linux-node1 nginx-1.10.3]# yum install gcc gcc-c++ zlib-devel pcre-devel openssl openssl-devel -y
[root@linux-node1 nginx-1.10.3]# ./configure --prefix=/usr/local/nginx-1.10.3 --user=www --group=www \ 
--with-http_ssl_module --with-http_stub_status_module --with-file-aio --with-stream
[root@linux-node1 nginx-1.10.3]# make && make install

# 开始配置

  • 简单配置测试下 nginx.conf 文件
worker_processes  1;
events {
    worker_connections  1024;
}
stream {  
        upstream tcp_proxy {
        hash $remote_addr consistent;  #远程地址做个hash
        server 192.168.1.4:22;
   }
      server {
        listen 2222;
        proxy_connect_timeout 1s;
        proxy_timeout 10s;  #后端连接超时时间
        proxy_pass tcp_proxy;
     }
  }

上面的配置是将本机的 2222 端口转发到 192.168.1.4 的 22 端口，配置之后，试验结果：通过了。

同理，开始配置数据库端口的转发。我们按照上面的配置，稍加改动下：

worker_processes  1;
events {
    worker_connections  1024;
}
stream {  
        upstream tcp_proxy {
        hash $remote_addr consistent;  #远程地址做个hash
        server 192.168.1.4:3306;
   }
      server {
        listen 3306;
        proxy_connect_timeout 1s;
       # proxy_timeout 10s;  #后端连接超时时间
        proxy_pass tcp_proxy;
     }
  }
