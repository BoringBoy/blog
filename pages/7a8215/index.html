# 磁盘空间管理

# 磁盘扩展

  • 以VMware中的Linux为例
  1. 关闭虚拟机
# 方式1
shutdown -h now

# 方式2
poweroff
1
2
3
4
5

  1. 打开VMware，虚拟机-->设置，选择内存，进行调整，点击确定即可。

  2. 启动虚拟机，查看磁盘情况

# 查看磁盘占用(非必要)
df -h
1
2
3

此处由于还没将扩展的磁盘空间进行分区，所以看到的空间占用信息跟设置前比较没有变化

  1. 查看磁盘分区信息
 fdisk -l
1

此处可以看到红色框选部分Disk部分信息已经发生了改变，磁盘大小包含扩展的空间了。

  1. 开始用命令对磁盘分区 (1)执行分区命令
fdisk /dev/sda
1

(2)输入'p',输出已有分区信息

(3) 依次输入下面的字符：

  • 输入n回车，新增一个分区；
  • 输入p回车，在打开的分区号命中中使用默认的分区号并回车（此处是3）
  • 在显示的起始扇区first sector直接回车，在last sector直接回车；
  • 输入p回车，查看当前分区的信息：可以看到多了一个/dev/sda3的新分区；
  • 输入w回车，写入磁盘信息并保存。

(4) 分区完成后，重启虚拟机(必须重启，否则/dev/sda4无法格式化)

(5) 格式化分区

# 格式化分区为ext4文件系统
sudo mkfs.ext4 /dev/sda4
1
2

(6) 对新分区进行挂载

# 为新分区创建一个挂载目录
mkdir /ext4

# 打开编辑特定文件
sudo vim /etc/fstab

# 文件末尾追加以下内容,指定要挂载的分区以及挂载点的相关信息
# 表示/dev/sda4分区开机自动挂载在/ext4目录下，文件系统类型为ext4，使用默认挂载参数
/dev/sda4   /ext4   ext4   defaults   0   0

# 接着保存文件

df -h

  1. "If you want to create more than four partitions, you must replace a primary partition with an extended partition first."
  • 问题分析：
    根据Chatgpt描述，MBR（Master Boot Record）分区表方案的限制。在 MBR 分区表中，最多可以有4个主分区，或者3个主分区和1个扩展分区.如果您已经有4个主分区，并且想创建更多的分区，您需要先将一个主分区转换为扩展分区。扩展分区可以容纳逻辑分区，从而允许您在一个扩展分区内创建更多的分区。

# 系统磁盘清理

# 磁盘占用排查

  1. 首先确定是否是磁盘满了
    命令：df -h

  • 参数说明

    -a：列出所有的文件系统，包括系统特有的/proc等文件系统

    -k：以KB的容量显示各文件系统

    -m：以MB的容量显示各文件系统

    -h：以人们较易阅读的GB,MB,KB等格式自行显示

    -H：以M=1000K替代M=1024K的进位方式

    -T：连同该分区的文件系统名称（例如ext3）也列出

    -i：不用硬盘容量，而以inode的数量来显示

  1. 显示每个目录的大小
    命令:du -sh /*

  • 参数说明
    -a : 列出所有的文件与目录容量，因为默认仅统计目录下面的文件量而已；

    -h : 以人们较易读的容量格式（G/M）显示；

    -s : 列出总量，而不列出每个个别的目录占用了容量；

    -S : 不包括子目录下的总计，与-s有点差别；

    -k : 以KB列出容量显示；

    -m : 以MB列出容量显示。

  1. 其它
  • 寻找当前目录，哪个文件夹占用空间最大
du -h --max-depth=1
1
  • 将文件以从大到小顺序展现
ls –lhS
1

# 常用

  1. 查看当前目录及其子目录的占用大小，并按占用空间大小进行排序：
du -h --max-depth=1 | sort -hr
# -h 参数用于以易读的方式显示文件大小
# --max-depth=1 参数用于限制只显示第一级子文件夹的信息。
1
2
3
  1. 查看指定目录及其子目录的占用大小，并按占用空间大小进行排序：
du -h --max-depth=1 /path/to/folder | sort -hr

# 将显示 /path/to/folder 及其子目录的占用大小，
# 并按照占用空间从大到小进行排序。
