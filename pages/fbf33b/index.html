# ubuntu/debian包管理

本篇文章整理记录了ubuntu/debian系统中包管理的相关概念和常用命令，并给出离线安装包时缺少依赖的几种解决办法。

# ubuntu/debian

# 基本知识

# 1. apt 与apt-get 区别？ 该如何选择？

apt 和 apt-get 都是 Ubuntu 系统中常见的包管理工具，用于软件的安装、更新和卸载。

(1)区别：

  • 命令行参数不完全相同：

    apt 命令行参数与 apt-get 不完全相同。例如，apt 中没有 apt-get 命令的 -y 选项，而是使用 --yes。

  • 进度条显示方式不同：

    apt 命令默认会显示进度条，而 apt-get 命令默认不会显示进度条。如果想要在 apt-get 命令中显示进度条，可以使用 -y 或 -q 等选项。

  • 自动清理方式不同： apt 默认会在安装、升级和清理软件包时自动清理无用依赖，而 apt-get 命令默认不会清理，需要使用单独的命令进行清理。

(2)选择根据：

  • Ubuntu 版本：

    如果使用 Ubuntu 16.04 或更新版本，则可以使用 apt 命令。如果是较旧的版本，则应该使用 apt-get 命令。

# 2. 相关文件存放路径

  • 软件源地址路径： /etc/apt/source.list

  • 软件包存放地址： /var/cache/apt/archives

# 常用命令

# apt 命令

#apt [选项] 命令
#更新可用软件包列表
apt update  

#通过 安装/升级 软件来更新系统
apt upgrade

#查找包
apt search [package]

#安装包
apt install [package]

#重新安装包
apt reinstall [package]

#移除包
apt remove [package]

#显示已安装的软件包
apt list --installed

#显示软件包具体信息例如：版本号，安装大小，依赖关系，bug报告等等
apt show [package]

# apt-cache命令

#获取依赖包信息
apt-cache depends [package]

#查看该包被哪些包依赖
apt-cache rdepends [package]

#搜索本地缓存的包
apt-cache search package_name
#只在软件包名称中查找搜索词
apt-cache search --names-only package_name

#软件包的详细元数据信息
apt-cache show package_name

  • apt的策略

默认情况下，每个已安装的软件包版本的优先级为 100，未安装的软件包的优先级为 500。同一软件包可能有多个不同优先级的版本。APT 会安装优先级较高的版本，除非安装的版本较新。

# 解决离线安装缺少依赖的问题

# 方法1：使用 aptitude 工具

下载后默认存放位置在/var/cache/apt/archives

  1. 安装工具
sudo apt install aptitude
1
  1. 下载指定软件包
# 执行下面命令
sudo aptitude --download-only install <package-name>
# 示例
sudo aptitude --download-only install vsftpd
1
2
3
4

# 方法2：使用apt-get命令

  1. 输入下面的命令
sudo apt-get –d install [package]
1

说明: 此方法仅用于当前系统中没有该包及其依赖时才有效，如果通过安装其他包时已经安装了某个依赖，这个方法就不会下载那个依赖项。（因此该方法大概率会漏项，即便是在新的系统上操作也是极大可能漏项的，因为操作系统安装过程中已经安装了一些包）

# 方法3：apt-get install

  • depends方式找出依赖项交由管道处理后传递给apt-get install命令
#有些包名中有<>符号，用tr将其删除
sudo apt-get install --reinstall -d `apt-cache depends [packname] | grep depends | cut -d: f2 |tr -d "<>"`

sudo dpkg -i *.deb
