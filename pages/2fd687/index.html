#查看所有索引
GET _cat/indices?v

#查看指定索引
GET indexName

# 文档管理

# 创建文档

# 可用put/post创建，其中put需要指定索引唯一id,post不用
put indexName/_doc/index_id
1
2

# 查询文档数据

# 根据索引id查询索引数据
GET indexName/_doc/index_id

# 查询所有索引数据
GET indexName/_search
1
2
3
4
5

# 修改文档数据

POST indexName/_doc/index_id
1
2

注意事项：修改时，要在原有基础上修改，而不能只指定单个key的修改，否则生效后，原先其它key会被舍弃。

即：如果原有key:id,name,age等，如果希望修改age,也需要将id,name 等Post过去修改，否则最后的数据只会保留age.

# 删除文档数据

DELETE indexName/_doc/index_id

# 分词查询

  • 简单条件
GET indexName/_search
{
    "Query":{
        "match": {
            "key" : "value"
        }
    }
}

此处会有分词匹配效果，假设key.value="hello world",则会将'hello' 和'world'分为两个词，即匹配目标文档中包含 key.value="hello"和key.value="world"的数据，而不会将"hello world"视为一个整体

  • 完整匹配
GET indexName/_search
{
    "Query":{
        "term":{
            "key":{
                "value" :"text"
            }
        }
    }
